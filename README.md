# Boss直聘 Agent 2.0

亲爱的面试官见字如面，感谢你打开这个git仓库，这是agent更新后的内容，

如果您是从 BOSS 直聘上看到我的简历，那么大概率是我通过这个 Agent 完成了岗位搜索与初步投递。

## 项目简介

Boss直聘 Agent 2.0 是上一版本的升级版。本版本从原来的视觉识别方案，切换为基于 **Chrome DevTools MCP** 的网页 DOM 分析方案。

相比 1.0 版本，本版本在运行稳定性、点击准确率、执行速度和页面跳转逻辑上都有明显改进。

---

## 2.0 版本更新内容

### 1. 不再依赖桌面视觉窗口

上一版本需要持续展示 Boss直聘 页面，并通过截图识别页面内容。

在 2.0 版本中，Agent 使用 **Chrome DevTools MCP** 直接获取网页 DOM 信息进行分析，因此不再强依赖桌面截图，也不需要始终将 Boss直聘 页面展示在最前方。



### 2. 提升页面操作稳定性

上一版本基于视觉模型，需要对页面截图进行缩放，再由 LLM 判断鼠标坐标并执行点击操作。

这种方式存在一定误差，尤其是在页面缩放、窗口尺寸变化、元素位置偏移时，容易出现点击不准确的问题。

2.0 版本改为基于 DOM 的元素识别与点击，点击准确率大幅提升。



### 3. 提升运行速度

视觉模型分析图片的速度相对较慢，尤其是在需要频繁截图和判断页面状态时，会明显影响整体执行效率。

本版本采用 `qwen3:8b` 纯文本模型进行页面状态分析，减少了视觉识别带来的额外开销，运行速度相比上一版本有明显提升。



### 4. 对于上一版本反爬虫手段的误解

其实boss直聘目前为止我没有遇到反爬问题。

上一版本采用视觉模型而不是文本模型是因为错误的认为about:blank状态是boss直聘的反爬

但是它只是ai编写代码时自动生成一个about:blank来作为中间态来输入新的url。

在本版权不再使用about:blank，而是直接在当前页面输入目标 URL，降低了页面跳转状态错乱的概率。

## 使用注意事项

### 1. 浏览器要求

当前版本仅支持 **Google Chrome 浏览器**，暂不支持其他浏览器。



### 2. 自动化权限

运行代码后，Chrome 浏览器可能会提示是否允许自动化控制。

请根据提示允许浏览器自动化操作，否则 Agent 可能无法正常运行。



### 3. 网络环境要求

使用时建议关闭代理或 VPN。

如果使用国外网络环境，或者浏览器默认搜索引擎不是 Bing，可能会导致 Boss直聘 页面加载速度变慢。

页面加载过慢时，可能会出现 URL 已变化但 DOM 尚未完成加载的情况，进而导致 Agent 对当前页面状态判断错误，甚至造成程序崩溃。

---

## 当前仍存在的问题

### 1. 暂不支持自动回复 Boss直聘 消息

当前版本暂时无法自动回复 Boss直聘 的消息。

该问题与上一版本一致，仍需要后续继续完善消息页面的状态机。



### 2. 对低性能环境支持仍不完善

当网络较慢、电脑性能较低或网页加载较慢时，Agent 仍可能出现 URL 与 DOM 状态不同步的问题。

例如：

* DOM 已经开始刷新，但页面关键元素尚未加载完成；

目前仍在探索更稳定的 URL 与 DOM 同步机制，以保证 Agent 能够在页面加载完成后再继续执行下一步操作。

---

## 后续优化方向

### 1. 增强 Boss直聘 页面状态机

后续计划继续完善 Boss直聘 的页面状态机，例如：

* 消息提示状态识别；
* 异常页面识别；

通过增加更多页面状态判断，让 Agent 在真实网页环境下运行得更加稳定。



### 2. 完善 URL 与 DOM 同步机制

后续会继续优化页面同步逻辑，确保 URL 和 DOM 之间能够互相等待。

目标是避免以下问题：

* DOM 已刷新，但关键元素暂未出现；

最终希望实现更加可靠的页面等待机制，让 Agent 能够根据真实页面状态做出判断，而不是单纯依赖固定等待时间。

---

## 总结

Boss直聘 Agent 2.0 相比上一版本，主要完成了以下升级：

* 从视觉识别切换为 DOM 分析；
* 使用 Chrome DevTools MCP 提升页面控制能力；
* 使用 `qwen3:8b` 纯文本模型提升分析速度；
* 优化页面跳转逻辑，避免 `about:blank` 中间态；
* 提高点击准确率和运行稳定性。

虽然当前版本仍然存在消息回复、慢网络环境适配、URL 与 DOM 同步等问题，但整体框架已经比 1.0 版本更加稳定，已经可以正式运行的一个工程。
